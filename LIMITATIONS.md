# LIMITATIONS

- Este MVP usa dados ficticios.
- O MVP nao reconhece cartas oficiais reais automaticamente.
- O MVP nao usa backend.
- O MVP nao envia imagens, camera ou dados de partida para servidores.
- O MVP depende da qualidade das imagens de referencia.
- O reconhecimento pode variar com iluminacao, reflexos, oclusao, distancia e angulo da camera.
- A biblioteca de image tracking e criada em runtime a partir de PNGs placeholder; em producao academica, validar tambem uma `XRReferenceImageLibrary` serializada no Editor.
- O ambiente atual nao testou build real no Samsung Galaxy S25 Ultra.
- O ambiente atual nao testou build real no iPhone 16 Pro Max.
- Build iOS precisa ser feito em macOS com Xcode.
- Feedback sonoro e haptico esta previsto/configuravel, mas ainda e simples.
- Gravacao de tela nao faz parte do MVP; pode ser futura somente com consentimento explicito.
- Os resultados estatisticos citados na proposta sao sinteticos e nao devem ser tratados como coleta real.
